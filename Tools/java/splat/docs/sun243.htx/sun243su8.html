3.5 DataLink - more information about the data

SPLAT-VO  supports DataLink and SODA (server-side manipulation through DataLink). SSAP and ObsCore services providing DataLink allows the user to access other resources available about the spectrum. If the service offers Datalink, by right-cliking on a Spectrum from either an ObsCore or SSAP query result list, there should be an option "DataLink" in the pop-up menu. When choosing this option, a new window will open with a list of links to resources about this data. This resources could be links to related publications, a preview image, or the dataset in other formats, or link to functions to perform server-side operations ( SODA) on the data (like cutouts, normalisation, etc). If there’s a link to a preview image then SPLAT will automatically display it on this window. If there are SODA services provided, the interactive formular to deal with the data is also displayed.

Server-side manipulation with DataLink/SODA

A query response table may contain descriptions of SODA services, which can be either defined specifically for single spectra, or for all of the returned spectra. In both cases an input formular will be produced by SPLAT-VO where parameters can be entered. In the first case, the formular appears in the Datalink resources list when right-clicking one line from the query results list. In the second case the soda operation can be submitted one or more selected spectra at once. This is done by clicking the Datalink button on the lower right side of the results table: After a SSAP or ObsCore query is sent, SPLAT-VO  checks the responding servers for SODA definitions.
Table-wide SODA operations are present, service will be marked with a symbol ( ). If at least one DataLink supporting server has been found, a DataLink button will appear below the query results. By clicking on it, DataLink. will be activated, and all results tabs will be deactivated except the results from servers that support DataLink. When you choose one of the active services, a window will appear, where the respective parameters for sever-side manipulation can be entered. The parameters will be set by clicking on the set parameters button. While dataLink is activated and the parameters are set, the spectra you select and display or download will be manipulated using the chosen parameters. To deactivate dataLink and have retrieve the original spectra just click on the dataLink button again.

The DataLink links window

The DataLink links window is created when the Datalink option at the popup menu obtained by right-clicking on a spectrum from either an ObsCore or SSAP query result list. It contains a table containing all the links this available for the spectrum. The table has one row for each link, and columns containing among other information the location, content-type, size, and semantics (what is in this link).
In the semantics column, #this means the actual spectrum, which would be displayed by SPLAT-VO when clicking on the spectrum in the query results list. If a #preview is available, it will be automatically displayed in the DataLink window. If there are SODA links (semantics #proc), the respective form will also be displayed below the links table. The presence of an error message means link is not available.
As many of the links can contain objects in data formats SPLAT-VO cannot understand (and are not necessarily spectra or timeseries), SPLAT-VO splat does not open all available links. If the content type is something that SPLAT-VO could understand, it tries to open as a spectrum. If the content type is text/html, the link will be opened in a web browser. The description column should tell if it makes sense to click on the link to open it in SPLAT-VO or not. By right-clicking on a link, it’s possible to copy the row, or just the access url to the clipboard, which can be used to open it with other application.
